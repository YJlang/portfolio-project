# 관심 연구실 후보와 확장 기준

이 문서는 포트폴리오가 특정 연구실 하나에 고정되지 않도록, 관심 연구실 후보와 평가 기준을 유동적으로 관리하기 위한 장기 업데이트 문서다. 앞으로 관심사가 변하거나 새로운 연구실을 발견하면 후보를 계속 추가한다.

## 현재 우선 관심 후보

| 상태    | 연구실/기관                            | 링크                        | 연결 관심사                              | 메모                                                         |
| ------- | -------------------------------------- | --------------------------- | ---------------------------------------- | ------------------------------------------------------------ |
| 탐색 중 | 국민대 Business IT 전문대학원 AIMS Lab | https://aims.kookmin.ac.kr/ | AIX, CX, Business IT, 고객 분석, AI 응용 | 현재 관심 후보. 포트폴리오 전체를 이 랩에만 맞추지는 않는다. |

## 확장 가능한 연구 방향

- AIX: AI가 사용자 업무와 의사결정을 보조하는 방식.
- CX / UX: 고객 경험, 페르소나, 여정 지도, 서비스 개선.
- HCI: 사람과 AI/소프트웨어 시스템의 상호작용, 인터페이스, 피드백.
- Business IT / MIS: 비즈니스 문제 해결을 위한 데이터, 소프트웨어, 운영 시스템.
- Agentic Software Development: MCP, Skill, LLM Agent, 개발 자동화.
- Web Security / DevOps: 신뢰성, 배포, 운영, 보안 관점의 웹 시스템.

## 연구실 후보 평가 기준

새로운 연구실을 찾을 때 아래 기준으로 메모한다.

| 기준            | 확인 질문                                                                                       |
| --------------- | ----------------------------------------------------------------------------------------------- |
| 연구 적합성     | 연구실 주제가 AIX, CX, HCI, UX, Business IT, Agentic Software Development 중 어디와 연결되는가? |
| 프로젝트 연결성 | PULSE, personaLab, BlogAuto 중 어떤 프로젝트가 가장 강한 근거가 되는가?                         |
| 방법론 적합성   | 정량 분석, UX 리서치, 프로토타이핑, AI 시스템 구현 중 어떤 역량을 강조해야 하는가?              |
| 진학/지원 전략  | 포트폴리오에서 어떤 섹션을 전면에 배치해야 하는가?                                              |
| 보완 필요 자료  | 논문, 실험, 데모 영상, 기술 문서, 회고 중 무엇을 추가해야 하는가?                               |

## 후보 추가 템플릿

```md
### 연구실명

- 링크:
- 기관/학과:
- 연구 키워드:
- 연결 프로젝트:
- 강하게 맞는 이유:
- 보완해야 할 포트폴리오 자료:
- 지원 시 강조 문장:
```

## 장기 관리 규칙

- 관심 후보는 확정/탈락으로 성급히 나누지 않고 `탐색 중`, `우선 관심`, `보류`, `지원 준비`, `아카이브` 상태로 관리한다.
- 연구실 이름보다 연구 키워드와 내 프로젝트의 연결성을 우선 기록한다.
- 특정 연구실에 맞춘 문장도 남기되, 포트폴리오 전체 문구는 넓은 연구 방향에 맞게 유지한다.
- 매 학기 또는 큰 프로젝트가 끝난 뒤 이 문서를 다시 보고 후보와 강조점을 업데이트한다.
