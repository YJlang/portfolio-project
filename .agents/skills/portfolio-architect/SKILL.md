# Portfolio Architect

Use this skill when working on this repository's portfolio strategy, content structure, project storytelling, research-fit sections, or long-term update system.

## Core Intent

Treat this portfolio as a lifelong, continuously updated professional knowledge base, not a one-time application page.

Optimize for:

- clear current identity
- durable structure for future projects, research, writing, awards, and career updates
- reusable source material for graduate school, jobs, freelance work, open source, and personal branding
- project case studies that explain judgment, not just output
- flexible research positioning across AIX, CX, HCI, UX/UI, Business IT, Web Security, DevOps, and Agentic Software Development

## Required Context

Before changing portfolio strategy or copy, read:

- `AGENTS.md`
- `docs/portfolio-blueprint.md`
- `docs/portfolio-references.md`
- `default-info.MD`
- `projects.MD`
- `target-labs.MD`

## Working Rules

- Do not lock the portfolio to a single lab, company, school, or short-term goal.
- Keep the homepage focused on the present: identity, representative work, and contact path.
- Move older but meaningful work into index or archive structures instead of deleting it by default.
- Make each major project able to grow into a case study with problem, role, system, UX/CX insight, result, and reflection.
- Prefer stable categories over temporary labels: `Featured`, `Research`, `Product`, `Backend`, `AI/Agent`, `Archive`.
- When adding new content, preserve date, source, link, public/private status, and future follow-up needs.

## Case Study Template

Use this structure for representative projects:

```md
## Project Name

### Problem

Who had what problem, friction, repeated work, or decision-making need?

### My Role

What did Junho own? Clarify personal contribution in team projects.

### System

How did the frontend, backend, AI/API layer, data, and deployment fit together?

### UX / CX Insight

What user, customer, or workflow judgment shaped the implementation?

### Result

What was built, validated, shipped, learned, or improved?
```

## Output Standard

For substantial portfolio planning work, produce:

- diagnosis of the current material
- proposed structure
- content priorities
- update or archive recommendations
- concrete files to change
- next-step checklist
